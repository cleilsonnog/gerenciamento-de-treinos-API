# 🚀 Task: Integração de Pagamentos com Stripe

## 🎯 Objetivo

Implementar sistema de assinatura utilizando Stripe, com controle de planos no backend e persistência no banco de dados.

---

## 🧱 Arquitetura Esperada

Fluxo completo:

Frontend (Next.js)
→ API (backend existente)
→ Stripe
→ Webhook (Stripe → API)
→ Banco de dados (atualização do usuário)

---

## 🔧 Etapas de Implementação

### 1. Configuração do Stripe

- Criar conta no Stripe
- Criar dois produtos:
  - Plano Mensal → R$ 14,90
  - Plano Trimestral → R$ 39,90

- Obter os `price_id` de cada plano

---

### 2. Backend (API)

#### Criar endpoint:

`POST /create-checkout-session`

Responsabilidades:

- Receber o usuário autenticado
- Receber o plano selecionado (mensal ou trimestral)
- Criar sessão de checkout no Stripe usando `price_id`
- Retornar `checkoutUrl` para o frontend

---

### 3. Banco de Dados

Adicionar campos na tabela de usuários:

```sql
plan: string (free | monthly | quarterly)
stripe_customer_id: string
subscription_id: string
subscription_status: string (active | canceled | incomplete | past_due)
```

---

### 4. Webhook (ESSENCIAL)

Criar endpoint:

`POST /webhook`

Responsabilidades:

- Receber eventos do Stripe
- Validar assinatura do webhook
- Tratar eventos importantes:

#### Eventos obrigatórios:

- `checkout.session.completed`
- `invoice.paid`
- `customer.subscription.updated`

#### Ações:

- Criar ou atualizar cliente no banco
- Salvar `stripe_customer_id`
- Atualizar plano do usuário
- Atualizar status da assinatura

---

### 5. Frontend (Next.js)

#### Na Landing Page:

- Botões de assinatura (mensal / trimestral)
- Ao clicar:
  - Chamar `/create-checkout-session`
  - Redirecionar para `checkoutUrl`

---

### 6. Redirecionamento pós-pagamento

Configurar no Stripe:

- `success_url` → redirecionar para `/login`
- `cancel_url` → retornar para landing page

---

## ⚠️ Regras Importantes

- ❌ NÃO usar secret do Stripe no frontend

- ❌ NÃO confiar apenas no redirect de sucesso

- ❌ NÃO liberar acesso baseado apenas no frontend

- ✅ SEMPRE usar webhook como fonte da verdade

- ✅ SEMPRE validar eventos do Stripe

- ✅ SEMPRE persistir no banco

---

## 🧠 Boas Práticas

- Usar variáveis de ambiente para:
  - STRIPE_SECRET_KEY
  - STRIPE_WEBHOOK_SECRET

- Garantir que endpoints são protegidos

- Associar usuário autenticado ao checkout

---

## 🚀 Extras (Opcional)

- Criar middleware para bloquear acesso sem plano ativo
- Criar gerenciamento de assinatura em @app/profile/page.tsx, sem alterar os dados que ja estão no banco, so adicione essa funcionalidade.
- Permitir cancelamento de plano

---

## ✅ Resultado Esperado

- Usuário escolhe plano na landing
- É redirecionado para Stripe
- Após pagamento:
  - Webhook atualiza banco
  - Usuário passa a ter plano ativo

- Sistema reconhece plano do usuário corretamente
